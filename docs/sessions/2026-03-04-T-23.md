  - `src/main/resources/static/admin/app.js`
  - `docs/api/openapi-selfhosted-byom-v1.yaml`
- Why this approach:
  - 기존 인증 체계(고정 토큰 맵)를 유지하면서는 토큰 수명주기와 운영자 권한 경계를 구현하기 어렵기 때문에, `TokenHashStore`를 lifecycle 저장소로 확장하고 `role` 기반 인가를 보안 필터 체인에 직접 연결했다.
  - 컨테이너 최초 onboarding 요구(AC1)를 위해 bootstrap API를 공개 경로로 두고, bootstrap 완료 후 운영자 토큰을 UI에 자동 주입해 `/admin` 단일 흐름을 완성했다.
  - 레거시 DB 호환 리스크를 줄이기 위해 `SqliteTokenHashStore`에 스키마 감지/마이그레이션을 추가했다.
