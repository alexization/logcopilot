  - `src/test/resources/admin-ui-runtime-check.js` - `app.js`를 stub DOM 위에서 실행하는 런타임 검증 스크립트다.
  - `src/test/java/com/logcopilot/connector/LokiConnectorEndpointsContractTest.java` - Loki 조회/secret 유지 contract를 검증한다.
  - `src/test/java/com/logcopilot/policy/PolicyEndpointsContractTest.java` - 정책 조회와 validation contract를 검증한다.
  - `src/test/java/com/logcopilot/alert/AlertEndpointsContractTest.java` - 알림 조회/secret 유지/min confidence contract를 검증한다.
- Why this approach:
  - T-24 핵심은 "저장"이 아니라 "기존값 조회 후 수정" UX이므로, 저장 API 재사용만으로는 완료할 수 없다.
  - 설정 섹션별 조회 GET을 명시적으로 제공하고 UI에서 섹션 진입 시 자동 로드하도록 해 재접속 시나리오를 직접 충족했다.
  - 미설정 상태도 `configured=false`로 표준화해 UI가 오류 대신 초기 입력 상태로 동작하도록 만들었다.
